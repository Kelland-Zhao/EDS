## 2026-06-04 Inspection2.0 点检记录合并：6 Sheet → 1

修改文件：`Code.js`

### 修改内容
- 新增 `USE_UNIFIED_INSPECTION_SHEET` 开关（紧急回滚设 false）
- `getData_PointCheck()`：Sheet 名从 `process + "-" + workshop` 改为 `"InspectionRecords"` + B/C 列过滤
- `getData_PointCheck_Inspection2()`：INJ-TB1/TB2 硬编码、TF/PK 动态表名、遍历 sheet 构建 allHistoricalRecords 三处统一改为读 InspectionRecords + 按工序过滤
- `upload_PointCheck()` / `submitInspectionResults()`：写入目标从动态 `process + "-" + workshop` 改为统一 `"InspectionRecords"`
- 新增 `migrateToInspectionRecords()` 一次性迁移函数 + `countInspectionRows()` 验证函数

### 原因
- 6 个 Sheet（INJ-TB1/TB2、TF-TB1/TB2、PK-TB1/TB2）列结构完全一致，B/C 列已含车间/工序
- 合并后查询更简单、新增工序/车间无需新建 Sheet、统计不需跨表聚合
- 减少 API 调用次数（原来 2-6 次变为 1 次）
