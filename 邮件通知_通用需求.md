# 通知邮件 通用需求模板 / Notification Email General Requirements

> 本文档用于定义各模块通知邮件的统一规范与具体需求。
> 适用模块：FailureReport、Inspection、PM_Task 等。

---

## 通用规范 / General Rules

### 语言 / Language
- 邮件标题、正文所有字段标签均中英双语

### 发送方式 / Sending
- 多个收件人合并发送：收集所有收件人后一次性调用 `GmailApp.sendEmail(to, subject, '', { htmlBody, cc })`，`to` 为第一个收件人，其余用逗号拼接填入 `cc`
- 所有测试函数收件人一律为 `kelland_zhao@colpal.com`

### 收件人规则 / Recipient Rules
- 操作方通知被操作方，操作方自己抄送（例：验证人操作 → `to`: 责任人，`cc`: 验证人）
- 汇总提醒类：按接收人邮箱分组，每人发一封，包含其名下所有待处理记录
- 收件人邮箱从 `名字【邮箱】` 格式中提取，正则：`/【([^】]+)】/`

### 样式规范 / Style
- 邮件格式为 HTML 富文本
- 表头背景色 `#E60012`、白色文字
- 数据行交替条纹（奇行 `#f5f5f5` / 偶行 `#ffffff`）
- 外层容器 `border-radius: 8px`、`box-shadow`
- 状态字段颜色：已通过 `#28a745`（绿）、未通过 `#dc3545`（红）、其他 `#6c757d`（灰）
- 页脚注明"此邮件由系统自动发送，请勿回复"（中英双语）

### 内容规范 / Content
- 日期格式统一：`YYYY-MM-DD`（使用 `Utilities.formatDate(date, tz, 'yyyy-MM-dd')`）
- 邮件必须包含：触发本次通知的核心数据字段 + 操作结果/新状态
- 操作类通知（非汇总提醒）需包含系统访问入口链接按钮
