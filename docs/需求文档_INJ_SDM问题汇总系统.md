# INJ SDM 问题汇总系统需求文档

## 1. 项目概述

在 EDS 中新增“INJ SDM 问题汇总”模块，供 INJ 工序管理员汇总过去一段连续日期内的大故障、当天遗留问题，以及需要在 SDM / Team 中沟通的事项。

系统支持记录新增、历史查询、编辑和删除，并保存最后修改时间及编辑历史。

## 2. 用户与权限

### 2.1 使用对象

- INJ 工序管理员。

### 2.2 权限来源

- 复用 EDS 权限表中的现有管理员权限，不新增权限列。
- 工序匹配 `INJ`，兼容 `IM`。
- Navigation 页面仅向有权限用户显示模块入口卡片。
- 无权限用户不显示入口卡片。
- 后端必须再次校验权限，防止通过 URL 直接访问或调用接口。

### 2.3 责任人候选

- 数据来源：权限表 `userID`。
- 工序列：O 列，值为 `INJ`，兼容 `IM`。
- 职位列：P 列，值为 `S&C`。
- 责任人支持多选。
- 正式页面加载时动态读取候选人，不在前端硬编码。

## 3. 核心业务规则

| 项目 | 规则 |
|---|---|
| 填报日期 | 系统当天，只读 |
| 每日唯一 | 同一填报日期只能存在一份有效记录 |
| 数据日期 | 连续日期范围，默认开始和结束日期均为前一天 |
| 填写人 | 不显示、不保存 |
| 大故障 / 大问题 | 支持多条 |
| 今日遗留问题 | 支持多条 |
| SDM / Team 沟通事项 | 支持多条自由文本 |
| 沟通事项责任人 | 必填，至少选择一人，支持多选 |
| 编辑 | INJ 管理员可编辑 |
| 删除 | INJ 管理员可删除，操作前二次确认 |
| 修改记录 | 保存最后修改时间及编辑历史，不记录修改人 |

## 4. 页面结构

### 4.1 页面名称

- 中文：INJ SDM 问题汇总
- 英文：INJ SDM Issue Summary
- 浏览器标题：`INJ SDM 问题汇总 | INJ SDM Issue Summary`

### 4.2 标签页

1. 填写记录 / New Record
2. 历史查询 / History

### 4.3 填写记录

#### 基础信息

| 字段 | 控件 | 规则 |
|---|---|---|
| 填报日期 | 日期，只读 | 自动取当天 |
| 数据开始日期 | 日期选择 | 默认前一天 |
| 数据结束日期 | 日期选择 | 默认前一天，不得早于开始日期 |

#### 大故障 / 大问题

每条包含：

- 车间：`TB1` / `TB2` 平铺单选按钮。
- 机台号。
- 问题描述。
- 删除按钮。

支持逐条新增和批量粘贴整理。

#### 今日遗留问题

字段及交互与“大故障 / 大问题”相同。

#### SDM / Team 沟通事项

每条包含：

- 沟通事项自由文本。
- 责任人平铺多选按钮。
- 删除按钮。

责任人选中后显示红底白字。

### 4.4 历史查询

查询条件仅包含：

- 填报开始日期。
- 填报结束日期。

列表字段：

| 字段 | 说明 |
|---|---|
| 填报日期 | 当天填报日期 |
| 数据日期 | 单日或连续日期范围 |
| 大故障数量 | 当前记录的大故障条数 |
| 遗留问题数量 | 当前记录的遗留问题条数 |
| 沟通事项数量 | 当前记录的沟通事项条数 |
| 操作 | 查看、编辑、删除 |

## 5. 批量粘贴解析

### 5.1 输入形式

每行生成一条记录，支持以下分隔方式：

```text
H2123 问题描述
H2123: 问题描述
H2123：问题描述
H2123 - 问题描述
```

### 5.2 车间识别

| 机台号开头 | 自动识别车间 |
|---|---|
| `E0`、`H1`、`V1`、`S1` | TB1 |
| `H2`、`V2` | TB2 |
| 其他 | 车间留空，由用户手动选择 |

识别前将机台号统一转换为大写。

无法解析机台号或描述的行不静默丢弃，必须通过双语警告弹窗提示。

## 6. 保存前数据校验

点击保存时必须执行以下校验：

1. 数据开始日期和结束日期均已填写。
2. 数据开始日期不得晚于数据结束日期。
3. 数据结束日期不得晚于填报日期。
4. 至少填写一条大故障、遗留问题或沟通事项。
5. 大故障及遗留问题必须填写：
   - 车间。
   - 机台号。
   - 问题描述。
6. 可自动识别车间的机台号必须与选择的车间一致。
7. 沟通事项必须填写事项内容。
8. 每条沟通事项必须至少选择一名责任人。

校验失败时：

- 禁止提交。
- 错误字段红色高亮。
- 页面集中显示错误摘要。
- 使用符合 EDS UI 规范的双语错误弹窗提示。

## 7. 通知与交互规范

- 使用 SweetAlert2。
- 禁止使用原生 `alert()` 和 `confirm()`。
- 标题和内容均为中英双语。
- 成功提示在 1500 ms 后自动关闭。
- 删除操作使用 Warning 类型二次确认。
- 加载数据时禁止点击弹窗外关闭，并隐藏确认按钮。

## 8. 移动端要求

- 表单字段在手机端纵向排列。
- 输入框字号至少 16px，避免手机浏览器自动缩放。
- 输入框和操作按钮触控高度至少 44px。
- 保存按钮在页面底部保持易触达。
- 不出现页面级横向滚动。
- 车间和责任人按钮适合单手点击。
- 详情弹窗在移动端采用底部面板形式。

## 10. MasterData 单表结构

每个事项占一行，同一份每日记录通过 `ReportID` 关联。

| 列 | 字段 | 说明 |
|---|---|---|
| A | ReportID | 同日记录唯一 ID |
| B | ReportDate | 填报日期，每日唯一 |
| C | DataStartDate | 数据开始日期 |
| D | DataEndDate | 数据结束日期 |
| E | ItemID | 事项唯一 ID |
| F | Category | `MAJOR` / `OUTSTANDING` / `COMMUNICATION` |
| G | Workshop | `TB1` / `TB2`；沟通事项可为空 |
| H | MachineNo | 机台号；沟通事项可为空 |
| I | Description | 问题或沟通事项内容 |
| J | OwnerNames | 责任人姓名展示文本 |
| K | OwnersJSON | 责任人数组 JSON |
| L | CreatedAt | 创建时间 |
| M | UpdatedAt | 最后修改时间 |
| N | Status | `ACTIVE` / `DELETED` |
| O | EditHistoryJSON | 编辑历史数组 JSON |

### 10.1 唯一性

- `ReportDate + Status=ACTIVE` 只能对应一个 `ReportID`。
- 保存前后端必须检查当日是否已有有效记录。
- 当日已有记录时执行更新，不新增第二份记录。

### 10.2 删除策略

- 删除采用逻辑删除，将相关行的 `Status` 更新为 `DELETED`。
- 删除前的数据写入 `EditHistoryJSON`。
- 历史查询默认只读取 `ACTIVE` 数据。

### 10.3 编辑策略

- 更新同一 `ReportID` 下的全部事项。
- 保存 `UpdatedAt`。
- 修改前快照追加到 `EditHistoryJSON`。
- 不保存修改人。

## 11. 后端接口建议

| 函数 | 用途 |
|---|---|
| `checkINJSDMPermission()` | 校验模块管理员权限 |
| `getINJSDMInitData()` | 获取权限、责任人候选及当天记录 |
| `saveINJSDMReport(payload)` | 新增或更新当天记录 |
| `getINJSDMHistory(startDate, endDate)` | 按填报日期查询历史 |
| `getINJSDMReport(reportId)` | 获取记录详情 |
| `deleteINJSDMReport(reportId)` | 逻辑删除记录 |
| `getINJSCUsers()` | 获取 INJ/IM 工序 S&C 候选人 |

所有新增、编辑和删除接口必须进行后端权限校验。

## 12. Navigation 集成

- 新增模块卡片。
- 建议图标：`bi-chat-square-text` 或 `bi-clipboard-data`。
- 仅有权限的 INJ 管理员显示卡片。
- 无权限用户不显示禁用占位卡片。
- 卡片跳转遵循 EDS 现有 URL 参数规则。

## 13. 验收标准

1. 有权限的 INJ 管理员可以看到 Navigation 入口。
2. 无权限用户看不到入口，直接访问 URL 也无法读取或修改数据。
3. 数据日期默认前一天，并支持连续日期范围。
4. 三类事项均支持多条录入和删除。
5. 沟通事项责任人从权限表动态加载并支持多选。
6. 批量粘贴可拆分多行，并按规则识别 TB1/TB2。
7. 不符合车间识别规则的机台号保留记录并要求手动选择。
8. 保存前校验完整，错误数据不能写入。
9. 同一填报日期不能生成两份有效记录。
10. 历史查询仅按填报日期范围筛选。
11. 管理员可以查看、编辑和删除历史记录。
12. 编辑后保存最后修改时间及修改前快照。
13. 页面在桌面端和移动端均无明显布局问题。
14. 所有通知符合 EDS UI 规范。
