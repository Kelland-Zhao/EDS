# EDS 任务安排模块设计文档

版本：V20260627.01  
日期：2026-06-27  
适用系统：EDS Equipment Data System  
模块名称：任务安排 / Daily Work Arrangement

## 1. 项目目标

任务安排模块用于解决设备部门每日人员安排和工作任务管理问题。模块作为 EDS 内部功能，不做独立登录页，复用 EDS 当前登录用户身份，并在 EDS 现有用户权限表中增加最小权限字段。

第一版重点解决：

| 目标 | 说明 |
|---|---|
| 每日人员管理 | 管理当天人员出勤状态、岗位、班次 |
| 工作任务管理 | 创建、分配、跟踪跨天持续任务 |
| 多人协作 | 支持主负责人和多个协作人 |
| 权限控制 | 管理员、主管、员工三类角色 |
| 通知提醒 | App 内提醒、企业微信群机器人、邮件通知 |
| 操作追溯 | 关键操作写入日志 |

## 2. 第一版范围

| 模块 | 第一版内容 |
|---|---|
| EDS 集成 | 作为 EDS 内部模块，从 EDS 菜单进入 |
| 身份识别 | 复用 EDS 当前登录用户身份 |
| 权限 | 在 `userID` 表新增 `任务安排权限` 字段 |
| 角色 | `admin` / `supervisor` / `employee` |
| 今日工作台 | 今日人员、岗位、今日任务、延续任务、超期任务 |
| 每日人员安排 | 从昨日复制、从模板生成、手动调整 |
| 任务管理 | 跨天任务、主负责人、协作人、状态流转 |
| 员工操作 | 新建个人任务、查看今日所有任务、更新自己相关任务 |
| 主管操作 | 生成每日安排、派工、调整/完成/取消全部任务 |
| 管理员操作 | 权限维护、配置维护、全部数据管理 |
| 通知 | App 内提醒、企微固定群、邮件通知 |
| 日志 | 写入任务模块日志 |

## 3. 第一版不做

| 暂不做 | 原因 |
|---|---|
| 统计报表 | 先跑通每日安排和任务执行 |
| 设备/模具关联 | 第一版任务只记录文字描述 |
| 个人企微通知 | 第一版使用固定群机器人 |
| 任务审批流 | 员工个人任务允许直接创建 |
| 绩效/负荷分析 | 后续基于任务数据扩展 |

## 4. EDS 权限复用设计

已通过 MCP 读取 EDS 用户权限表：

| 项目 | 内容 |
|---|---|
| Spreadsheet | `UserID` |
| 主表 | `userID` |
| 员工主数据 | `EffectiveEmployee` |
| 日志表 | `EditLog` |

`userID` 表为两行表头结构：

| 行 | 用途 |
|---|---|
| 第 1 行 | 模块分类，例如 `EDS`、`Tasklist_MoC` |
| 第 2 行 | 字段名，例如 `SAPID`、`NAME`、`邮箱`、`项目跟进权限管理` |

### 4.1 复用字段

| 字段 | 用途 |
|---|---|
| `SAPID` | 用户主键 |
| `NAME` | 姓名 |
| `邮箱` | 匹配 EDS 当前登录用户 |
| `车间` | 所属车间 |
| `工序` | 所属工序 |
| `班别` / `班次` | 班组、班次信息 |
| `直线上级邮箱 / Supervisor Email` | 超期任务邮件 CC，可选 |

### 4.2 新增字段

第一版只新增 1 个权限字段：

| 第 1 行模块 | 第 2 行字段 | 值 | 说明 |
|---|---|---|---|
| `EDS` | `任务安排权限` | 空 / `admin` / `supervisor` / `employee` | 空值表示无权限 |

### 4.3 权限识别流程

| 步骤 | 逻辑 |
|---|---|
| 1 | EDS 获取当前登录用户邮箱 |
| 2 | 到 `userID` 表按 `邮箱` 匹配用户 |
| 3 | 读取 `SAPID`、`NAME`、`车间`、`工序`、`班别`、`任务安排权限` |
| 4 | 权限为空时不显示任务安排模块入口 |
| 5 | 权限非空时进入对应角色页面 |

## 5. 页面结构

| 页面 | 主要功能 |
|---|---|
| EDS 任务安排入口 | 从 EDS 菜单进入模块 |
| 今日工作台 | 日期选择、人员安排、今日任务、延续任务、超期提醒 |
| 每日安排页 | 从昨日复制、按模板生成、调整出勤状态和岗位 |
| 任务列表页 | 按状态、负责人、日期筛选跨天任务 |
| 任务详情/编辑弹窗 | 新建、编辑、分配负责人和协作人、更新状态、备注 |
| 我的任务页 | 当前用户负责/协作的未完成任务 |
| 模板配置页 | 维护每日安排模板 |
| 配置页 | 维护企微、邮件、提醒规则 |
| 操作日志页 | 管理员/主管查看关键操作记录 |

电脑端首页采用工作台布局：

| 区域 | 内容 |
|---|---|
| 顶部栏 | 日期选择、当前用户、角色 |
| 左侧 | 今日人员：姓名、出勤状态、岗位、备注 |
| 中间 | 今日任务：未开始、进行中、等待中 |
| 右侧 | 超期任务、异常任务、无人负责任务 |
| 弹窗 | 新建任务、编辑任务、查看日志 |

## 6. 业务流程

| 步骤 | 操作人 | 动作 |
|---|---|---|
| 1 | 管理员 | 在 `userID` 表维护 `任务安排权限` |
| 2 | 管理员/主管 | 维护岗位模板、通知配置 |
| 3 | 主管 | 选择日期，从昨日复制或从模板生成每日安排 |
| 4 | 主管 | 调整人员出勤、岗位、班次 |
| 5 | 主管/员工 | 新建任务 |
| 6 | 主管 | 分配主负责人和协作人 |
| 7 | 员工 | 查看今日所有任务，更新自己相关任务 |
| 8 | 主管 | 跟进全部任务，必要时代完成或取消 |
| 9 | 系统 | 写入提醒、发送企微和邮件、记录日志 |

任务不是每天清空，而是持续存在。今日工作台按日期筛选需要关注的任务，包括今日新建、今日计划开始、今日截止、未完成延续任务和超期任务。

## 7. 数据表结构

### 7.1 `DailyStaff`

| 字段 | 说明 |
|---|---|
| `DailyStaffID` | 唯一 ID |
| `Date` | 日期，YYYY-MM-DD |
| `SAPID` | 人员工号，关联 `userID` |
| `AttendanceStatus` | 在岗 / 休息 / 请假 / 外出 / 培训 |
| `WorkRole` | 当日岗位，如设备、模具、SAP、点检 |
| `Shift` | 班次 |
| `Remark` | 备注 |
| `CreatedAt` | 创建时间 |
| `CreatedBy` | 创建人 SAPID |
| `UpdatedAt` | 更新时间 |
| `UpdatedBy` | 更新人 SAPID |

### 7.2 `Tasks`

| 字段 | 说明 |
|---|---|
| `TaskID` | 唯一任务编号 |
| `Title` | 任务标题 |
| `Description` | 任务描述 |
| `TaskType` | 日常 / 临时 / 延续 |
| `Priority` | 高 / 中 / 低 |
| `Status` | 未开始 / 进行中 / 等待中 / 已完成 / 已取消 |
| `PlanStartDate` | 计划开始日期 |
| `DueDate` | 截止日期 |
| `CompletedAt` | 完成时间 |
| `CreatedBy` | 创建人 SAPID |
| `ClosedBy` | 完成人 SAPID |
| `Remark` | 备注 |
| `CreatedAt` | 创建时间 |
| `UpdatedAt` | 更新时间 |

### 7.3 `TaskMembers`

| 字段 | 说明 |
|---|---|
| `TaskMemberID` | 唯一 ID |
| `TaskID` | 关联任务 |
| `SAPID` | 参与人 |
| `MemberRole` | `owner` / `collaborator` |
| `CanUpdate` | `Y` / `N`，第一版默认 `Y` |
| `CreatedAt` | 创建时间 |
| `CreatedBy` | 创建人 SAPID |

### 7.4 `DailyTemplates`

| 字段 | 说明 |
|---|---|
| `TemplateID` | 模板 ID |
| `TemplateName` | 模板名称，如白班模板 |
| `SAPID` | 模板内人员 |
| `DefaultAttendanceStatus` | 默认出勤状态 |
| `DefaultWorkRole` | 默认岗位 |
| `DefaultShift` | 默认班次 |
| `SortOrder` | 展示顺序 |
| `Active` | `Y` / `N` |

### 7.5 `TaskNotifications`

| 字段 | 说明 |
|---|---|
| `NotificationID` | 提醒 ID |
| `SAPID` | 接收人 |
| `TaskID` | 关联任务，可空 |
| `Type` | 新任务 / 今日任务 / 即将到期 / 超期 |
| `Title` | 提醒标题 |
| `Content` | 提醒内容 |
| `ReadStatus` | 未读 / 已读 |
| `CreatedAt` | 创建时间 |

### 7.6 `TaskLogs`

| 字段 | 说明 |
|---|---|
| `LogID` | 日志 ID |
| `Action` | 操作类型 |
| `TargetType` | `Task` / `DailyStaff` / `Template` |
| `TargetID` | 目标 ID |
| `BeforeValue` | 修改前 JSON |
| `AfterValue` | 修改后 JSON |
| `OperatorSAPID` | 操作人 SAPID |
| `OperatorName` | 操作人姓名 |
| `CreatedAt` | 操作时间 |

### 7.7 `TaskConfig`

| 字段 | 说明 |
|---|---|
| `Key` | 配置键 |
| `Value` | 配置值 |
| `Remark` | 备注 |
| `UpdatedAt` | 更新时间 |
| `UpdatedBy` | 更新人 SAPID |

建议配置项：

| Key | Value 示例 |
|---|---|
| `EnableEmailAlert` | `Y` |
| `EnableWeComAlert` | `Y` |
| `WeComWebhookUrl` | 企业微信群机器人 Webhook |
| `DailySummaryTime` | `08:00` |
| `DueSoonDays` | `1` |
| `OverdueCcSupervisor` | `Y` |

## 8. 权限规则

| 功能 | 管理员 | 主管 | 员工 |
|---|---:|---:|---:|
| 查看今日工作台 | Y | Y | Y |
| 查看今日所有任务 | Y | Y | Y |
| 查看全部历史任务 | Y | Y | 仅自己相关 |
| 生成每日安排 | Y | Y | N |
| 修改出勤/岗位 | Y | Y | N |
| 维护模板 | Y | Y | N |
| 新建任务 | Y | Y | Y |
| 分配主负责人 | Y | Y | 只能设自己 |
| 添加协作人 | Y | Y | Y |
| 修改任务内容 | Y | Y | 仅自己创建且未完成 |
| 更新任务状态 | Y | Y | 仅自己相关 |
| 完成任务 | Y | Y | 仅自己相关 |
| 取消任务 | Y | Y | N |
| 查看操作日志 | Y | Y | N |
| 维护任务安排权限 | Y | N | N |

员工可以查看今日所有任务，方便团队协作，但只能修改自己创建、负责或协作的任务。

## 9. 任务状态

| 状态 | 说明 |
|---|---|
| 未开始 | 任务已创建，尚未开始 |
| 进行中 | 正在处理 |
| 等待中 | 等配件、等停机、等外部支持 |
| 已完成 | 任务结束 |
| 已取消 | 不再执行，仅管理员/主管可操作 |

## 10. 通知规则

第一版包含三类通知渠道：

| 渠道 | 用途 |
|---|---|
| App 内提醒 | 个人待办、超期、参与任务 |
| 企业微信群机器人 | 每日汇总、高优先级、超期异常 |
| 邮件通知 | 新任务、个人每日摘要、到期/超期提醒 |

### 10.1 App 内提醒

| 场景 | 接收人 |
|---|---|
| 新任务分配 | 主负责人、协作人 |
| 今日任务 | 主负责人、协作人 |
| 即将到期 | 主负责人、协作人 |
| 已超期 | 主负责人、协作人 |

### 10.2 企业微信提醒

| 场景 | 是否发送 | 说明 |
|---|---:|---|
| 每日任务汇总 | Y | 固定时间发到企微固定群 |
| 高优先级新任务 | Y | `Priority = 高` 时发送 |
| 超期任务 | Y | 每日汇总中列出，必要时单独发送 |
| 普通新任务 | N | 避免刷屏 |
| 普通状态变更 | N | 只写日志 |
| 任务完成 | N | 只在 App 内显示 |

### 10.3 邮件提醒

| 场景 | 是否发送 | 收件人 |
|---|---:|---|
| 新任务分配 | Y | 主负责人、协作人 |
| 高优先级任务 | Y | 主负责人、协作人、主管可选 |
| 即将到期 | Y | 主负责人、协作人 |
| 已超期 | Y | 主负责人、协作人，主管可选 CC |
| 每日任务汇总 | Y | 当天有任务人员 |
| 任务完成 | N | 不发 |
| 普通状态变更 | N | 不发 |

邮件地址来源：

| 字段 | 用途 |
|---|---|
| `userID`.`邮箱` | 任务接收人 |
| `userID`.`直线上级邮箱 / Supervisor Email` | 超期任务 CC，可选 |
