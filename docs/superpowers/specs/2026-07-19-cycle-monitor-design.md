## 1. 概述

新增「注塑周期监控 / Cycle Monitor」模块，以散点图小多图（Small Multiples）方式可视化 27 台注塑机台的单次平均周期数据，帮助注塑工序管理人员和工程师快速发现各机台的周期异常。

**核心用户**: 注塑工序管理人员、工程师

## 3. 可视化设计

### 3.1 图表类型

- **ECharts 散点图**（Scatter Chart），与资源甘特图共用同一图表库
- **小多图网格布局**：每台机台一张迷你散点图，按车间分组，Tab 切换

### 3.2 坐标轴

| 轴 | 变量 | 说明 |
|----|------|------|
| X 轴 | 数据日期 | 格式 `MM-DD`，无拥挤全显 |
| Y 轴 | 平均周期(秒) | 每台机台独立 Y 轴范围 = [标准-5, 标准+8] |

### 3.3 数据点

- **正常点**: 灰色 `#95A5A6`，圆形，大小 8px
- **异常点**: 红色 `#E74C3C`，圆形，大小 10px

### 3.4 异常判定规则

| 条件 | 判定 |
|------|------|
| 实际值 > 标准 + 3 秒 | **偏高异常** |
| 实际值 < 标准 - 1 秒 | **偏低异常** |
| 其他 | 正常 |

### 3.5 参考线

每张迷你图绘制三条水平参考线：

| 线名 | 颜色 | 线型 | 值 |
|------|------|------|-----|
| 上限线 | `#E74C3C` 红色 | 短虚线 `[5,5]` | 标准 + 3s |
| 标准线 | `#3498DB` 蓝色 | 实虚线 `[8,3]` | 标准周期 |
| 下限线 | `#F39C12` 橙色 | 短虚线 `[5,5]` | 标准 - 1s |

### 3.6 图例

底部统一图例：● 正常点 / ● 异常点 / ─ 标准线 / --- 上下限线

## 7. 后端函数

均在 `Code.js` 中新增。

### 7.1 `getCycleMonitorMachineList()`

- **入参**: 无
- **逻辑**: 读取 `机台周期标准` Sheet，按机台号前缀 H1→TB1 / H2→TB2 分组
- **返回**: `{ TB1: ["H1HTA755", ...], TB2: ["H2HTA520", ...] }`

### 7.2 `getCycleMonitorStandards()`

- **入参**: 无
- **逻辑**: 读取 `机台周期标准` Sheet，返回机台号→标准周期映射
- **返回**: `{ "H1HTA755": 30, "H1HTA756": 30, ... }`

### 7.3 `getCycleMonitorData(machines, days)`

- **入参**:
  - `machines`: `string[]` — 要加载的机台号数组
  - `days`: `number` — 往前推的天数（7 或 30）
- **逻辑**:
  1. 计算日期范围：`TODAY - days` ~ `TODAY`
  2. 从 `机台周期实际值` Sheet 读取全部数据
  3. 按日期范围过滤
  4. 按机台分组
  5. 匹配标准周期，逐行判定异常（>标准+3 或 <标准-1）
  6. 构造 JSON 返回
- **返回**:
```json
{
  "machines": [
    {
      "name": "H1HTA755",
      "workshop": "TB1",
      "standard": 30,
      "anomalyCount": 5,
      "points": [
        {
          "date": "07-13",
          "shift": "早班",
          "cycle": 31.2,
          "anomaly": false
        },
        {
          "date": "07-13",
          "shift": "中班",
          "cycle": 34.5,
          "anomaly": true,
          "deviation": 4.5
        }
      ]
    }
  ]
}
```

## 8. 前端数据流

```
页面加载
  │
  ├─ google.script.run.getCycleMonitorMachineList()
  │    └─ 渲染 TB1/TB2 Tab 标签
  │
  ├─ google.script.run.getCycleMonitorStandards()
  │    └─ 缓存标准值映射
  │
  └─ google.script.run.getCycleMonitorData(allMachines, 7)
       │
       └─ 按 workshop 分组
          ├─ 渲染当前激活 Tab 的小多图网格
          ├─ 统计异常机台数 → Tab Badge
          └─ 绑定点击放大事件

用户切换 Tab → 切换 visible 的网格区域（无需重新请求）
用户切换天数 → 重新调用 getCycleMonitorData(allMachines, newDays)
```

## 9. 边界情况

| 场景 | 处理方式 |
|------|----------|
| 某机台无数据（新机台/停产） | 网格格位显示空状态文字「暂无数值」 |
| 某日期仅部分班别有数据 | 有多少点画多少，不补齐空白 |
| 标准周期值缺失 | 不画三条参考线，图上方标注「标准值缺失」 |
| 首次加载数据量（27台×7天×3班≈567点） | 后端一次返回，前端渲染 < 2 秒 |
| 手机端访问 | 网格缩为 2 列，基本可用 |
| 数据源 Sheet 被移动/改名 | 后端 catch，返回 `{ error: "数据源不可用" }`，前端展示友好提示 |
| 服务账号无权访问 Sheet | 部署前确认 `sap-sheet-uploader@pyreadsp.iam.gserviceaccount.com` 已被授权读取该 Spreadsheet |
